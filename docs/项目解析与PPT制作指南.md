## 一、项目一句话描述

**CreatorChain** 是一个**基于区块链的数字创作版权保护平台**，通过创新的**双重确权机制**，为AI生成、人工创作、混合创作等所有类型的数字内容提供永久、可信、可验证的版权保护。

### 核心卖点（供PPT封面/开篇使用）

- 🎨 **多元创作支持** - 首个同时支持AI创作、手工创作、混合创作的版权平台
- 🔐 **双重确权机制** - 创新的两步确权流程，记录过程+确认结果
- 🛡️ **隐私保护** - 零知识证明技术，验证真实性同时保护创作隐私
- 💾 **永久存储** - IPFS去中心化存储，内容永不丢失
- 🤖 **AI深度集成** - 集成智谱、OpenAI等10+主流AI模型

## 二、项目背景与痛点

### 2.1 行业背景

随着AI技术的爆发式发展，2025年AI生成内容(AIGC)市场规模已达数千亿，预计2030年将突破万亿。然而，AI创作带来的版权问题日益严峻。

### 2.2 核心痛点（可做成问题→方案对照图）

| 痛点                       | 具体表现                               | 影响               |
| -------------------------- | -------------------------------------- | ------------------ |
| **版权归属不明确**   | AI创作、人机协作作品的版权归属存在争议 | 创作者权益无法保障 |
| **创作过程难以验证** | 传统版权登记无法证明创作的真实过程     | 版权纠纷难以裁定   |
| **AI创作确权困难**   | 现有法律和技术对AI创作版权保护存在空白 | AI创作者缺乏保护   |
| **收益分配不公平**   | 中间商抽取大量佣金，创作者收益微薄     | 打击创作积极性     |
| **存储安全隐患**     | 集中式存储存在数据丢失和篡改风险       | 作品可能永久丢失   |
| **跨平台确权难**     | 不同平台间版权信息无法互认             | 维权成本高昂       |

### 2.3 市场需求数据（可用于PPT图表）

- 全球数字内容创作者超过 **5000万**
- AI生成内容年增长率超过 **300%**
- 版权纠纷案件年均增长 **40%**
- 创作者因版权问题年损失超过 **100亿美元**

### 5.2 核心功能详解

#### 功能1：多元化创作支持

- **AI创作**：集成10+主流AI模型（智谱GLM、OpenAI、Stable Diffusion等）
- **手工创作**：支持上传已有数字作品
- **混合创作**：AI辅助+人工修改的组合创作

#### 功能2：双重确权机制 ⭐（核心创新点）

```
第一次确权（创作过程记录）          第二次确权（最终作品确认）
        ↓                                  ↓
┌─────────────────────┐          ┌─────────────────────┐
│ recordCreationProcess│    →    │ confirmFinalCreation│
├─────────────────────┤          ├─────────────────────┤
│ • 创作过程IPFS哈希   │          │ • 最终作品IPFS哈希  │
│ • 内容哈希           │          │ • 最终内容哈希      │
│ • 标题/描述          │          │ • 贡献度评分        │
│ • 创作类型(AI/手工)  │          │                     │
│ • 时间戳证明         │          │ • 确认时间戳        │
└─────────────────────┘          └─────────────────────┘
        ↓                                  ↓
   Stage 0: 过程已记录             Stage 1: 最终已确认
                                           ↓
                              ┌─────────────────────┐
                              │      mintNFT        │
                              ├─────────────────────┤
                              │ • 元数据URI         │
                              │ • 版税设置          │
                              │ • NFT铸造完成       │
                              └─────────────────────┘
                                           ↓
                                  Stage 2: NFT已铸造
```

#### 功能3：零知识证明隐私保护

- **Schnorr签名算法**：证明拥有秘密数据而不泄露内容
- **承诺方案**：Hash(原始数据+随机盐) → 链上存储
- **选择性披露**：用户可选择公开的信息级别

#### 功能4：IPFS分布式存储

- **内容寻址**：基于内容哈希的永久链接
- **多重备份**：本地节点+Pinata企业服务
- **防篡改**：内容一旦上传无法修改

#### 功能5：智能合约自动化

- **自动确权**：无需人工干预的确权流程
- **自动分成**：授权收入按比例自动分配
- **透明可查**：所有交易链上可验证

#### 功能6：积分激励系统

- **积分获取**：创作、互动、任务完成
- **积分消费**：购买内容、解锁功能
- **合规设计**：使用积分而非代币，符合监管

### 7.2 四大智能合约

#### 合约1: CreatorToken (CRT) - 平台代币

- **功能**：ERC-20代币，用于支付、质押、激励
- **特色**：通缩机制（每次转账燃烧0.5%）
- **代币分配**：
  - 创作奖励池 30% (3亿CRT)
  - 质押奖励 20% (2亿CRT)
  - 团队 15% (1.5亿CRT)
  - 社区空投 10% (1亿CRT)
  - 流动性 25% (2.5亿CRT)

#### 合约2: CreatorNFT - 版权NFT

- **功能**：ERC-721 + ERC-2981版税标准
- **核心**：实现双重确权机制
- **关键方法**：
  - `recordCreationProcess()` - 第一次确权
  - `confirmFinalCreation()` - 第二次确权
  - `mintNFT()` - 铸造NFT

#### 合约3: LicenseManager - 版权授权

- **功能**：管理版权授权和交易
- **授权类型**：
  - Personal (个人使用) - 1x价格
  - Commercial (商业使用) - 5x价格
  - Exclusive (独家授权) - 20x价格
  - OpenSource (开源授权) - 0.5x价格
- **自动分成**：平台2.5%，其余按比例分配给创作者

#### 合约4: CreatorDAO - DAO治理

- **功能**：去中心化自治组织
- **提案生命周期**：Pending → Active → Succeeded → Queued → Executed
- **治理参数**：
  - 投票延迟：1天
  - 投票期限：7天
  - 执行延迟：2天
  - 提案门槛：10000 CRT
  - 法定人数：100000 CRT

## 八、用户使用流程

### 8.1 主要用户流程（可做成流程图）

#### 流程1：AI创作并确权

```
1. 用户连接钱包登录
         ↓
2. 选择AI模型，输入提示词
         ↓
3. AI生成作品（系统自动记录过程）
         ↓
4. 第一次确权：记录创作过程到区块链
         ↓
5. 用户确认/修改作品
         ↓
6. 第二次确权：确认最终作品到区块链
         ↓
7. 铸造NFT，获得版权证书
         ↓
8. 可选择上架市场或保留
```

#### 流程2：手工作品确权

```
1. 用户连接钱包登录
         ↓
2. 上传已有数字作品（图片/文档等）
         ↓
3. 填写作品信息（标题/描述/类型）
         ↓
4. 作品上传到IPFS获得哈希
         ↓
5. 一键确权到区块链
         ↓
```

## 十三、常见问题预案

### Q1: 为什么需要双重确权？一次确权不够吗？

**答**：传统确权只记录最终作品，无法证明创作过程。双重确权先记录创作过程（AI模型、提示词、参数等），再确认最终作品，形成完整的创作证据链，在版权纠纷时可以证明创作的真实性和时间优先。

### Q2: 零知识证明具体是怎么实现的？

**答**：我们采用Schnorr签名+SHA256承诺方案。创作时，系统将秘密数据（提示词、参数等）哈希后上链，原始数据保存在本地。验证时，用户提供原始数据，系统重新计算哈希对比验证。这样既能证明拥有秘密数据，又不泄露具体内容。

### Q3: AI创作的版权到底归谁？

**答**：我们的贡献度评估算法量化了人类在AI创作中的贡献（提示词设计30%、参数优化25%、迭代调整20%等）。高贡献度说明人类创作者投入了大量创意劳动，版权归属更倾向于创作者。

### Q4: 为什么用积分而不用代币？

**答**：考虑到监管合规性，我们采用积分制度。积分可以在平台内使用，但不涉及金融交易，符合相关法律法规，同时也避免了区块链交易的高额Gas费用。

### Q5: IPFS存储会不会丢失数据？

**答**：我们采用多层存储策略：本地IPFS节点+Pinata企业级Pinning服务+备用节点。Pinata保证数据永久保存，即使本地节点故障也不会丢失。

### Q6: 智能合约安全性如何保证？

**答**：1) 使用OpenZeppelin审计过的安全库；2) 实现重入攻击防护(ReentrancyGuard)；3) 基于角色的访问控制；4) 时间锁机制防止恶意操作；5) 紧急暂停功能应对突发情况。

### Q7: 这个项目和现有的NFT平台有什么区别？

**答**：1) 现有NFT平台只关注结果，我们关注过程+结果；2) 我们专门为AI创作设计了贡献度评估；3) 零知识证明保护创作隐私；4) 双重确权机制是业界首创。
