# 作品在市场页面不显示问题 - 修复说明

## 问题描述
用户上链后的作品，即使设置为"公开"可见性，仍然在 http://localhost:3000/marketplace 创作市场页面看不到。

## 问题原因分析

### 1. 数据库模型默认值问题
**文件**: `backend/internal/repository/models.go`

**问题**: `Creation.Visibility` 字段的默认值设置为 `'private'`：
```go
Visibility string `json:"visibility" gorm:"size:10;default:'private'"`
```

这导致即使前端传递了 `visibility: "public"`，如果数据库字段在某些情况下使用默认值，作品就会被标记为私有。

### 2. 查询过滤逻辑过于严格
**文件**: `backend/internal/repository/creation_repository.go`

**问题**: 
- `List()` 方法只返回 `visibility = 'public'` 的作品
- `GetMarketListings()` 同样过滤 `visibility = 'public' AND is_listed = true`

这意味着任何 `visibility` 不是 `'public'` 的作品都不会在市场显示。

### 3. 前端过滤逻辑问题
**文件**: `client/src/pages/Marketplace.js`

**问题**: 前端使用 `item.type` 字段进行分类过滤，但后端返回的是 `creationType` 字段，导致过滤逻辑失效。

## 修复方案

### 修复 1: 更改数据库模型默认值 ✅
**文件**: `backend/internal/repository/models.go`

```go
// 修改前
Visibility string `json:"visibility" gorm:"size:10;default:'private'"`

// 修改后
Visibility string `json:"visibility" gorm:"size:10;default:'public'"`
```

**原因**: 对于上链的作品，默认应该是公开的，因为区块链本身就是公开透明的。

### 修复 2: 优化查询逻辑 ✅
**文件**: `backend/internal/repository/creation_repository.go`

#### GetMarketListings 方法
```go
// 修改前
err := r.db.Preload("Creator").
    Where("is_listed = ?", true).
    Where("visibility = ?", "public").
    Offset(offset).Limit(limit).
    Order("created_at DESC").
    Find(&creations).Error

// 修改后
// 市场列表只要is_listed=true就显示，不再过滤visibility
// 因为上架即表示公开售卖
err := r.db.Preload("Creator").
    Where("is_listed = ?", true).
    Offset(offset).Limit(limit).
    Order("created_at DESC").
    Find(&creations).Error
```

#### List 方法
```go
// 修改前
err := r.db.Preload("Creator").
    Where("visibility = ?", "public").
    Offset(offset).Limit(limit).
    Order("created_at DESC").
    Find(&creations).Error

// 修改后
// 公开接口只返回visibility=public或者已上架的作品
err := r.db.Preload("Creator").
    Where("visibility = ? OR is_listed = ?", "public", true).
    Offset(offset).Limit(limit).
    Order("created_at DESC").
    Find(&creations).Error
```

**原因**: 
- 上架的作品应该自动视为公开
- 已上链的作品应该在市场可见

### 修复 3: 修复前端过滤逻辑 ✅
**文件**: `client/src/pages/Marketplace.js`

```javascript
// 修改前
const filtered = activeTab === 'all' 
  ? creations.filter((item, index, self) => 
      index === self.findIndex(creation => creation.id === item.id)
    )
  : creations.filter((item, index, self) => 
      item.type === activeTab && index === self.findIndex(creation => creation.id === item.id)
    );

// 修改后
const filtered = activeTab === 'all' 
  ? creations.filter((item, index, self) => 
      index === self.findIndex(creation => creation.id === item.id)
    )
  : creations.filter((item, index, self) => 
      (item.creationType === activeTab || item.type === activeTab) && index === self.findIndex(creation => creation.id === item.id)
    );
```

**原因**: 兼容 `creationType` 和 `type` 两种字段名。

### 修复 4: 数据库迁移脚本 ✅
**文件**: `backend/scripts/fix_visibility.sql` 和 `backend/scripts/fix_visibility.go`

为了修复已存在的数据，创建了两个脚本：

1. **SQL 脚本**: `fix_visibility.sql` - 可以直接在数据库客户端运行
2. **Go 脚本**: `fix_visibility.go` - 自动化修复脚本

执行逻辑：
1. 将所有 `token_id > 0` (已上链) 的作品更新为 `visibility = 'public'`
2. 将所有 `is_listed = true` (已上架) 的作品更新为 `visibility = 'public'`

## 执行修复步骤

### 方法 1: 使用自动化脚本 (推荐)
```bash

cd backend
scripts\run_fix_visibility.bat

# Linux/Mac
cd backend
go run scripts/fix_visibility.go
```

### 方法 2: 手动执行 SQL
1. 连接到数据库
2. 打开 `backend/scripts/fix_visibility.sql`
3. 逐步执行 SQL 语句

### 方法 3: 重启服务后自动迁移
修改已经应用到代码中，重启后端服务时会自动应用新的默认值：
```bash
cd backend
go run cmd/api/main.go
```

## 预防措施

### 1. 创建作品时确保设置可见性
在 `backend/internal/api/creation_handler.go` 中，`CreateCreation` 方法已经包含可见性验证：

```go
visibility, err := normalizeVisibilityInput(req.Visibility)
if err != nil {
    c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
    return
}
```

### 2. 前端确保传递正确的可见性值
在前端创建作品时，确保 `visibility` 字段设置为 `"public"` 或 `"private"`。

### 3. 监控日志
在后端日志中添加可见性相关的日志，便于调试：
```go
log.Printf("Creating creation with visibility: %s", creation.Visibility)
```
