# 修复作品重复显示问题

## 问题描述

在"我的作品"页面中，同一个作品会显示两次：
- 一个版本可以正常显示（有图片、完整信息）
- 另一个版本无法正常显示（缺少图片或信息不完整）

用户明明只上传了一次作品，却出现了两个重复的条目。

## 问题根源

### 1. 数据流程问题

**ManualCreation.js 创建作品时的流程：**

```javascript
1. 调用 apiService.createCreation() → 保存到数据库
   - 此时作品有数据库 ID
   - 但还没有 token_id（因为还没上链）

2. 上传到区块链 → 获得 token_id
   
3. 尝试调用 apiService.updateCreationStatus() 
   - ❌ 问题：这个接口不存在！
   - 导致数据库中的作品没有 token_id

4. 保存到 localStorage
   - 包含完整信息（ID、token_id、图片等）
```

**MyCreations.js 加载作品时的流程：**

```javascript
1. 从后端获取作品
   - 有数据库 ID
   - ❌ 没有 token_id（因为没有更新成功）
   - 可能缺少图片URL

2. 从 localStorage 获取作品
   - 有数据库 ID
   - 有 token_id
   - 有完整的图片信息

3. 合并数据：[...后端作品, ...localStorage作品]
   
4. 去重逻辑使用：`${token_id}|${hash}` 作为key
   - ❌ 问题：后端作品没有 token_id，导致 key 不同
   - 结果：两个版本都被保留了
```

### 2. 去重逻辑缺陷

原来的去重逻辑：

```javascript
const key = `${tokenKey}|${hashKey}`;
```

**问题：**
- 只依赖 `token_id` 和 `hash`
- 如果后端作品缺少 `token_id`，会被识别为不同作品
- 没有优先考虑数据库 ID（最可靠的唯一标识）

## 解决方案

### 1. 修复后端 UpdateCreation 接口

**文件：** `backend/internal/api/creation_handler.go`

**改动：** 允许更新 `token_id` 和 `metadata_hash`

```go
var req struct {
    TokenID         *int64  `json:"token_id"`         // 新增
    MetadataHash    string  `json:"metadata_hash"`    // 新增
    Title           string  `json:"title"`
    Description     string  `json:"description"`
    Visibility      *string `json:"visibility"`
    AIModel         string  `json:"ai_model"`
    PromptText      string  `json:"prompt_text"`
    PriceInPoints   *int64  `json:"price_in_points"`
    LicenseDuration *int    `json:"license_duration"`
}

// 更新逻辑中添加
if req.TokenID != nil && *req.TokenID > 0 {
    creation.TokenID = *req.TokenID
}
if req.MetadataHash != "" {
    creation.MetadataHash = strings.TrimSpace(req.MetadataHash)
}
```

### 2. 修复 ManualCreation 更新逻辑

**文件：** `client/src/pages/ManualCreation.js`

**改动：** 使用正确的接口更新 token_id

```javascript
// 第八阶段：更新数据库状态（包括token_id）
if (backendCreationRecord?.id) {
  try {
    // 使用 updateCreation 接口更新完整信息，包括 token_id
    await apiService.updateCreation(backendCreationRecord.id, {
      token_id: registrationResult.creationId,
      metadata_hash: metadataHash,
      visibility: 'public'
    });
    console.log('数据库状态更新成功，包括token_id:', registrationResult.creationId);
  } catch (dbError) {
    console.debug('后端服务不可用，数据已保存到区块链:', dbError.message);
  }
}
```

### 3. 改进前端去重逻辑

**文件：** `client/src/pages/MyCreations.js`

**改动：** 多级去重策略 + 数据完整度评分

```javascript
// 改进的去重逻辑：优先使用数据库ID，其次使用token_id和hash
const dedupedMap = new Map();
const idSet = new Set(); // 跟踪已使用的数据库ID
const tokenIdSet = new Set(); // 跟踪已使用的token_id

for (const c of finalCreationsRaw) {
  const dbId = c.id ? Number(c.id) : null;
  const tokenId = c.token_id || c.TokenID;
  const hashKey = normalizeHash(c.content_hash || c.ContentHash || c.hash || c.fileHash || c.ipfsHash);
  
  // 计算数据完整度分数（用于选择最优版本）
  const completenessScore = [
    c.image || c.image_url,
    c.title && c.title !== '未命名作品',
    c.description,
    c.fileHash || c.content_hash || c.hash,
    c.creationType,
    c.category
  ].filter(Boolean).length;
  
  let shouldAdd = false;
  let key = '';
  
  // 优先级1: 使用数据库ID去重（最可靠）
  if (dbId && dbId > 0) {
    if (!idSet.has(dbId)) {
      key = `db_${dbId}`;
      idSet.add(dbId);
      shouldAdd = true;
    } else if (dedupedMap.has(`db_${dbId}`)) {
      // 如果ID已存在，比较完整度，保留更完整的版本
      const existing = dedupedMap.get(`db_${dbId}`);
      if (completenessScore > existing.score) {
        key = `db_${dbId}`;
        shouldAdd = true;
      }
    }
  }
  // 优先级2: 使用token_id去重（来自区块链）
  else if (tokenId) {
    const tokenKey = String(tokenId).trim();
    if (!tokenIdSet.has(tokenKey)) {
      key = `token_${tokenKey}`;
      tokenIdSet.add(tokenKey);
      shouldAdd = true;
    } else if (dedupedMap.has(`token_${tokenKey}`)) {
      const existing = dedupedMap.get(`token_${tokenKey}`);
      if (completenessScore > existing.score) {
        key = `token_${tokenKey}`;
        shouldAdd = true;
      }
    }
  }
  // 优先级3: 使用hash去重（最后的手段）
  else if (hashKey) {
    key = `hash_${hashKey}`;
    if (!dedupedMap.has(key)) {
      shouldAdd = true;
    } else {
      const existing = dedupedMap.get(key);
      if (completenessScore > existing.score) {
        shouldAdd = true;
      }
    }
  }
  
  if (shouldAdd && key) {
    dedupedMap.set(key, { creation: c, score: completenessScore });
  }
}

const finalCreations = Array.from(dedupedMap.values()).map(entry => entry.creation);
```

### 4. 清理本地存储重复数据

**文件：** `client/src/pages/MyCreations.js`

**改动：** 在加载时自动清理 localStorage 中的重复数据

```javascript
// 1.1 清理本地存储中的重复数据（基于ID去重）
if (localCreations.length > 0) {
  const localMap = new Map();
  localCreations.forEach(creation => {
    const key = creation.id || creation.token_id;
    if (key && !localMap.has(key)) {
      localMap.set(key, creation);
    }
  });
  localCreations = Array.from(localMap.values());
  // 保存清理后的数据
  localStorage.setItem('userCreations', JSON.stringify(localCreations));
  console.log('本地创作数据（清理后）:', localCreations.length);
}
```

## 修复效果

### 修复前
- ❌ 同一作品显示两次
- ❌ 其中一个版本缺少图片或信息
- ❌ 数据库中的作品没有 token_id
- ❌ 去重逻辑不够健壮

### 修复后
- ✅ 每个作品只显示一次
- ✅ 显示数据最完整的版本
- ✅ 数据库正确记录 token_id
- ✅ 多级去重策略，按 ID → token_id → hash 优先级
- ✅ 自动清理 localStorage 中的重复数据
- ✅ 优先保留数据完整度高的版本

4. **数据完整性**：优先保留数据最完整的版本
