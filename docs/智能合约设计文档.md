# CreatorChain 智能合约设计文档

## 📋 概述

CreatorChain 是一个基于区块链的数字创作版权保护平台，通过智能合约实现去中心化的版权确权、授权和交易。

**区块链技术点**:
```solidity
// 转账燃烧机制
function transfer(address to, uint256 amount) public override returns (bool) {
    uint256 burnAmount = (amount * burnRate) / 10000;
    uint256 transferAmount = amount - burnAmount;
    
    if (burnAmount > 0) {
        _burn(msg.sender, burnAmount);
    }
    return super.transfer(to, transferAmount);
}

// 质押奖励计算
function _calculateReward(address user) internal view returns (uint256) {
    uint256 duration = block.timestamp - info.startTime;
    uint256 lockBonus = 100 + (info.lockDuration * 100 / 365 days);
    return (info.amount * duration * rewardPerSecond * lockBonus) / (10**18 * 100);
}
```

### 2. CreatorNFT - 版权NFT

**文件（当前实现路径）**: `contracts/contracts/CreatorNFT.sol`

**核心功能**:
- **双重确权机制** ⭐ 项目亮点
  1. 第一次确权：记录创作过程
  2. 第二次确权：确认最终作品
- ERC-721 + ERC-2981 版税标准
- 防伪溯源验证
- 转移历史追踪

**双重确权流程**:
```
创作开始                     创作完成                    NFT铸造
   │                           │                          │
   ▼                           ▼                          ▼
┌─────────────┐          ┌─────────────┐          ┌─────────────┐
│recordCreation│  ───►   │confirmFinal │  ───►    │ mintNFT     │
│   Process   │          │  Creation   │          │             │
├─────────────┤          ├─────────────┤          ├─────────────┤
│• 过程IPFS哈希│          │• 最终IPFS哈希│          │• 元数据URI  │
│• 内容哈希    │          │• 内容哈希    │          │• 版税设置   │
│• 标题/描述   │          │• 贡献度评分  │          │             │
│• 创作类型    │          │              │          │             │
│• 时间戳      │          │• 时间戳      │          │             │
└─────────────┘          └─────────────┘          └─────────────┘
      │                        │                        │
      ▼                        ▼                        ▼
  Stage: 0               Stage: 1                 Stage: 2
  ProcessRecorded        FinalConfirmed           NFTMinted
```

**关键代码**:
```solidity
// 第一次确权
function recordCreationProcess(
    string memory processIpfsHash,
    bytes32 processContentHash,
    string memory title,
    string memory description,
    uint256 creationType  // 0=AI创作, 1=手工创作
) external returns (uint256) {
    // 检查内容唯一性，防止抄袭
    require(contentHashToTokenId[processContentHash] == 0, "Content already registered");
    
    // 创建记录
    info.processIpfsHash = processIpfsHash;
    info.processContentHash = processContentHash;
    info.processTimestamp = block.timestamp;  // 时间戳证明
    info.stage = CreationStage.ProcessRecorded;
    
    emit ProcessRecorded(tokenId, msg.sender, processIpfsHash, processContentHash, block.timestamp);
    return tokenId;
}

// 第二次确权
function confirmFinalCreation(
    uint256 tokenId,
    string memory finalIpfsHash,
    bytes32 finalContentHash,
    uint256 contributionScore  // 创作者贡献度 0-1000
) external {
    require(info.originalCreator == msg.sender, "Not the creator");
    require(info.stage == CreationStage.ProcessRecorded, "Invalid stage");
    
    info.finalIpfsHash = finalIpfsHash;
    info.finalContentHash = finalContentHash;
    info.contributionScore = contributionScore;
    info.confirmTimestamp = block.timestamp;
    info.stage = CreationStage.FinalConfirmed;
    
    emit FinalConfirmed(tokenId, msg.sender, finalIpfsHash, finalContentHash, contributionScore, block.timestamp);
}
```

function _processPayment(uint256 tokenId, uint256 amount) internal {

**提案生命周期**:
```
Pending ─► Active ─► Succeeded ─► Queued ─► Executed
   │          │          │
   │          │          └── 未达法定人数 ──► Defeated
   │          │
   │          └── 取消 ──► Canceled
   │
   └── 投票延迟期
```

**治理参数**:
| 参数 | 默认值 | 说明 |
|------|--------|------|
| votingDelay | 1天 | 提案创建后多久开始投票 |
| votingPeriod | 7天 | 投票持续时间 |
| executionDelay | 2天 | 通过后多久可执行 |
| proposalThreshold | 10000 CRT | 提案创建门槛 |
| quorumVotes | 100000 CRT | 法定投票人数 |
| passRate | 50% | 通过所需赞成比例 |

## 📊 技术亮点（期末答辩要点）

### 1. 双重确权创新
- **问题**: 传统版权登记无法证明创作过程
- **解决**: 两阶段上链，记录创作过程+最终成果

### 2. 通缩代币经济
- **机制**: 每次转账燃烧0.5%
- **价值**: 减少流通量，提升代币价值

### 3. 去中心化治理
- **DAO投票**: 社区决定平台发展方向
- **时间锁**: 防止恶意提案快速执行

### 4. 智能分成
- **自动执行**: 授权收入按比例自动分配
- **透明可验证**: 链上记录所有分成

### 5. 跨合约协作
- Token ←→ NFT ←→ License ←→ DAO
- 展示智能合约组合能力
