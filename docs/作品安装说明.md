# CreatorChain 作品安装说明

## 系统要求

### 硬件要求

- **CPU**: 4 核心以上 (推荐 8 核心)
- **内存**: 8GB 以上 (推荐 16GB)
- **存储**: 20GB 可用空间 (推荐 50GB)
- **网络**: 稳定的互联网连接 (推荐 100Mbps+)

### 软件要求

- **操作系统**: Windows 10/11, macOS 10.15+, Ubuntu 20.04+
- **Node.js**: 18.0+ (推荐 18.x LTS)
- **Go**: 1.21+（推荐使用 1.21.x，与后端代码保持一致）
- **Git**: 2.0+
- **数据库**: PostgreSQL 14+ 或 SQLite 3.0+（默认开发环境使用 SQLite，生产可切换 PostgreSQL）
- **Redis**: 6+ (可选，支持内存缓存)
- **IPFS**: 本地节点或 Pinata API

## 快速安装

### 1. 获取项目代码

可以直接使用 Git 克隆，也可以使用课堂提供的压缩包。

```bash
git clone https://github.com/Ylim314/CreatorChain.git
cd CreatorChain
```

### 2. 安装依赖

#### 安装前端依赖

```bash
cd client
npm install
cd ..
```

#### 安装后端依赖

```bash
cd backend
go mod tidy
go mod download
cd ..
```

#### 安装智能合约依赖

```bash
cd contracts
npm install
cd ..
```

### 3. 环境配置

#### 创建环境变量文件

```bash
# 在backend目录下创建.env文件
cd backend
cp .env.example .env
```

#### 配置环境变量

```bash
# .env文件内容
# 数据库配置（默认使用 SQLite，本地开发零依赖）
DATABASE_URL=sqlite:///creatorchain.db

# 或使用 PostgreSQL（推荐线上/演示环境）
# DATABASE_URL=postgres://user:password@localhost:5432/creatorchain?sslmode=disable

# Redis 配置（可选）
REDIS_URL=redis://localhost:6379

# 区块链配置（本地链或测试网）
# 本地开发通常使用 Hardhat/Ganache，本字段用于连接测试网或主网
ETHEREUM_RPC_URL=https://sepolia.infura.io/v3/your_key
PRIVATE_KEY=your_private_key

# AI 引擎配置
AI_API_KEY=your_ai_api_key
AI_BASE_URL=https://api.openai.com/v1

# IPFS 配置
IPFS_GATEWAY=https://ipfs.io/ipfs/
PINATA_API_KEY=your_pinata_api_key
PINATA_SECRET_KEY=your_pinata_secret_key

# 服务配置
PORT=8080
```

### 4. 数据库初始化

#### 使用 SQLite（默认）

- 无需额外配置，程序启动时会自动在 `backend` 目录下创建数据库文件（例如 `creatorchain.db`）。

#### 使用 PostgreSQL（可选）

```bash
# 创建数据库
createdb creatorchain

# 或使用 psql:
# psql -c "CREATE DATABASE creatorchain;"

# 程序启动时会自动运行数据库迁移
```

### 5. 启动服务

#### 方法一：使用批处理脚本（推荐）

**一键启动整个项目（Windows）**：

```bash
# 在项目根目录下运行
启动项目.bat
```


#### 方法二：手动启动

**启动后端服务**：

```bash
cd backend
go mod tidy
go run cmd/api/main.go
```

**启动前端服务**（新终端窗口）：

```bash
cd client
npm start
```

#### 编译智能合约 (可选)

```bash
cd contracts
npx hardhat compile
```

## 详细安装步骤

### 前端安装详解

#### 1. Node.js 环境准备

```bash
# 检查Node.js版本
node --version
npm --version

# 如果版本过低，请升级到最新LTS版本
# Windows: 从官网下载安装包
# macOS: brew install node
# Ubuntu: curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
```

#### 2. 前端依赖安装

```bash
cd client

# 安装生产依赖
npm install

# 安装开发依赖 (如果需要)
npm install --save-dev
```

#### 3. 前端配置

```bash
# 创建前端环境变量文件
cp .env.example .env.local

# 配置API地址
echo "REACT_APP_API_URL=http://localhost:8080" >> .env.local
echo "REACT_APP_CHAIN_ID=5" >> .env.local  # Goerli测试网
```

### 后端安装详解

#### 1. Go 环境准备

```bash
# 检查Go版本
go version

# 如果版本过低，请升级到1.24+
# Windows: 从官网下载安装包
# macOS: brew install go
# Ubuntu: sudo apt install golang-go
```

#### 2. 后端依赖安装

```bash
cd backend

# 下载依赖
go mod tidy
go mod download

# 验证依赖
go mod verify
```

#### 3. 数据库配置

##### SQLite 配置（推荐新手）

```bash
# 无需额外配置，会自动创建数据库文件
# 数据库文件位置取决于 DATABASE_URL 通常在 backend 目录下
```

##### PostgreSQL 配置（推荐生产/演示环境）

```bash
# 确保已经创建数据库 creatorchain
# 编辑 backend/.env 文件，将 DATABASE_URL 改为类似：
DATABASE_URL=postgres://user:password@localhost:5432/creatorchain?sslmode=disable
```

### 智能合约安装详解

#### 1. 安装 Hardhat 依赖

```bash
cd contracts

# 验证安装
npx hardhat --version
```

#### 2. 编译合约

```bash
# 编译所有合约

# 运行测试
npx hardhat test
```

## 部署说明

### 开发环境部署

#### 1. 本地开发

```bash
# 启动所有服务
npm run dev  # 如果配置了脚本

# 或者分别启动
# 终端1: 后端服务
cd backend && go run cmd/api/main.go

# 终端2: 前端服务
cd client && npm start
```

#### 2. 访问应用

- **前端**: http://localhost:3000
- **后端 API**: http://localhost:8080
- **API 文档**: http://localhost:8080/docs

### 生产环境部署

#### 1. 前端部署

```bash
cd client

# 构建生产版本
npm run build

# 部署到静态服务器
# 将build目录内容上传到Web服务器
```

#### 2. 后端部署

```bash
cd backend

# 编译二进制文件
go build -o creatorchain-api cmd/api/main.go

# 运行服务
./creatorchain-api
```

#### 3. 智能合约部署

本项目当前演示主要使用 `SimpleCreationRegistry` 进行本地确权联调：

```bash
cd contracts

# 编译合约
npx hardhat compile

# 部署到本地链（Hardhat 或其他在 localhost:8545 的节点）
npx hardhat run scripts/deploy-simple.js --network localhost

# 如需部署到公共测试网（例如 Sepolia/Polygon Mumbai），
# 请在 hardhat.config.cjs 中配置对应 RPC 和私钥后，指定网络名称：
# npx hardhat run scripts/deploy-simple.js --network sepolia
```

## 常见问题解决

### 1. 端口冲突

```bash
# 检查端口占用
netstat -ano | findstr :3000
netstat -ano | findstr :8080

# 杀死占用进程
taskkill /PID <进程ID> /F
```

### 2. 依赖安装失败

```bash
# 清理缓存
npm cache clean --force
go clean -modcache

# 重新安装
rm -rf node_modules package-lock.json
npm install
```

### 3. 数据库连接失败

```bash
# 检查数据库服务状态
# MySQL
sudo systemctl status mysql

# SQLite (无需服务，检查文件权限)
ls -la backend/creatorchain.db

# 检查连接参数
# 确认.env文件中的数据库URL正确
```

### 4. 区块链连接失败

```bash
# 检查网络连接
ping polygon-rpc.com

# 检查API密钥
# 确认.env文件中的POLYGON_RPC_URL有效
```

### 2. 功能测试

#### 基础功能测试

1. **用户注册/登录**
   - 访问前端页面 http://localhost:3000
   - 连接MetaMask钱包
   - 完成用户注册
   - 验证积分奖励 (1000积分)

2. **双重确权流程测试**
   - **第一次确权**: 记录创作过程
     - 上传作品文件（AI生成或人工创作）
     - 填写创作信息（AI模型信息或创作工具）
     - 记录创作过程（提示词和参数或创作步骤）
     - 生成Merkle树根
   - **第二次确权**: 确认最终作品
     - 确认作品内容
     - 生成验证证明
     - 记录到区块链
     - 获得版权证书

3. **多元化创作功能**
   - **AI生成创作**: 使用AI模型生成内容，智能分析贡献度
   - **手工创作**: 上传传统手工制作的作品，记录创作过程
   - **混合创作**: AI辅助的人工创作，记录双重创作贡献
   - **内容存储**: 自动上传到IPFS，获得永久分布式存储

4. **市场功能**
   - 浏览作品列表
   - 购买作品 (使用积分)
   - 查看交易记录
   - 管理收藏夹
